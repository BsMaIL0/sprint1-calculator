# 📟 Консольный калькулятор с памятью

[![C++](https://img.shields.io/badge/C++-11-blue.svg)](https://en.cppreference.com/w/cpp/11)
[![CMake](https://img.shields.io/badge/CMake-3.10+-green.svg)](https://cmake.org/)

Консольный калькулятор с поддержкой арифметических операций, памяти (save/load) и обработкой ошибок. Учебный проект в рамках курса Яндекс Практикум.

## ✨ Возможности

- **Базовые операции**: `+`, `-`, `*`, `/`
- **Возведение в степень**: `**`
- **Управление памятью**: `s` (сохранить), `l` (загрузить)
- **Команды**: 
  - `=` — вывод текущего значения
  - `:` `<число>` — установить значение
  - `c` — сброс в 0
  - `n` — смена знака
  - `q` — выход
- **Обработка ошибок**: неверные команды, отсутствие числа в памяти, деление на ноль
- **Поддержка научной нотации**: `1e-5`, `1.5e10`

## 📖 Команды

| Команда | Описание | Пример |
|---------|----------|--------|
| `+ N` | Прибавить N | `+ 5` |
| `- N` | Вычесть N | `- 3` |
| `* N` | Умножить на N | `* 4` |
| `/ N` | Разделить на N | `/ 2` |
| `** N` | Возвести в степень N | `** 3` |
| `=` | Вывести текущее значение | `=` |
| `: N` | Установить значение N | `: 42` |
| `c` | Сбросить в 0 | `c` |
| `n` | Сменить знак | `n` |
| `s` | Сохранить в память | `s` |
| `l` | Загрузить из памяти | `l` |
| `q` | Выход | `q` |

## 🧪 Обработка ошибок

| Ситуация | Сообщение |
|----------|-----------|
| Неверное число | `Error: Numeric operand expected` |
| Неизвестная команда | `Error: Unknown token <token>` |
| Загрузка из пустой памяти | `Error: Memory is empty` |
| Деление на ноль | `inf` (бесконечность) |

## 🏗️ Архитектура

```
┌─────────────────┐     ┌─────────────────┐
│    main.cpp     │────▶│  CalculatorUI   │
│  (точка входа)  │     │  (UI + парсер)  │
└─────────────────┘     └────────┬────────┘
                                 │
                                 ▼
                        ┌─────────────────┐
                        │   Calculator    │
                        │ (бизнес-логика) │
                        └─────────────────┘
```


**Ключевые решения:**

- Разделение логики (`Calculator`) и UI (`CalculatorUI`)
- Использование `using Number = double` для гибкости смены типа
- Работа с потоками через ссылки (`std::ostream&`, `std::istream&`)
- Сквозная обработка ошибок с возвратом `bool`

## 📊 Тестирование

Проект содержит встроенные тесты в `main.cpp`:

```cpp
std::istringstream input{"42 / 6 + 3 = q"};
CalculatorUI ui{calc, output, errors};
ui.Parse(input);
assert(calc.GetNumberRepr() == "10.000000");
```

Примеры из задания покрывают 11 сценариев (корректные вычисления, ошибки ввода, работа с памятью).

## 📝 Уроки, вынесенные из проекта

1. **Разделение ответственности** — калькулятор не должен знать про `s` и `l`, это задача UI
2. **Тестируемость** — передача потоков ссылками позволяет легко мокать ввод/вывод
3. **Обработка ошибок** — важно не только сообщить об ошибке, но и корректно завершить работу
4. **Типобезопасность** — `using Number` вместо голого `double` упрощает будущие изменения
